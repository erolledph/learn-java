## 2026-04-07 - Cache curriculum navigation indices
**Learning:** The application's global `state` in `app.js` tracked navigation context via `currentLesson` and `currentModule` but lacked their respective indices (`currentLessonIndex`, `currentModuleIndex`). This led to inefficient O(N) nested array lookups (e.g., `findIndex`) in `previousLesson`, `nextLesson`, and `updateNavigationButtons` across the entire `JAVA_CURRICULUM` on every navigation event.
**Action:** When tracking global state for nested list items, always cache their indices alongside the objects to enable O(1) state lookups and avoid redundant traversals.
## 2024-04-08 - Remove redundant JS syntax highlighting to fix typing performance
**Learning:** Found an anti-pattern where a custom JS function (`applySyntaxColors`) was traversing the DOM and applying inline styles to CodeMirror elements on every keystroke (`change` event). This was redundant because `styles.css` already properly defined the syntax highlighting rules. Furthermore, the `setTimeout` was not debounced, queueing massive layout thrashing operations on rapid keystrokes, completely blocking the main thread.
**Action:** When working with established UI libraries (like CodeMirror), trust their styling mechanisms (CSS classes) rather than manually querying and styling the DOM elements dynamically with JS, especially on high-frequency events like typing.
